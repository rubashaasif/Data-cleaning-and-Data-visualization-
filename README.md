# Google Play Store Data Cleaning and Visualization

## Overview
This project focuses on cleaning, preprocessing, and analyzing the Google Play Store dataset. Through exploratory data analysis (EDA) and visualizations, the project uncovers trends in app ratings, installs, categories, pricing, and user engagement.

## Objectives
- Clean and preprocess raw Google Play Store data
- Handle missing values and inconsistent formats
- Perform exploratory data analysis (EDA)
- Create visualizations to identify trends and patterns
- Generate insights about app performance and popularity

## Technologies Used
- Python
- Pandas
- NumPy
- Matplotlib
- Seaborn
- Jupyter Notebook / Google Colab

## Dataset
The dataset contains information about applications available on the Google Play Store, including:
- App Name
- Category
- Rating
- Reviews
- Size
- Installs
- Price
- Content Rating
- Genres

## Project Workflow
1. Data Loading
2. Data Cleaning
   - Handling missing values
   - Removing duplicates
   - Converting data types
3. Exploratory Data Analysis
4. Data Visualization
5. Insights and Conclusions

## Key Visualizations
- Distribution of App Ratings
- Top Categories by Number of Apps
- Installs Analysis
- Reviews vs Ratings
- Free vs Paid Apps Comparison
- Category-wise Trends

## Key Insights
- Most apps on the Play Store are free.
- Certain categories dominate in app count and installs.
- Ratings generally cluster around higher values.
- Popular apps tend to receive significantly more reviews.
